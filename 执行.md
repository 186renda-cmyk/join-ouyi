Trae，关于 Clean URL 的标准化修复，我发现还有一个非常关键的地方被遗漏了：【结构化数据 (Schema/JSON-LD)】。

为了彻底消除 .html 后缀带来的重定向风险，请将修复范围扩大到全站所有代码。

🔥 最终完整的执行清单 (请严格逐项检查):

1. 修复 Canonical 标签 (SEO 核心)

检查 <link rel="canonical">。

规则： href 必须是无后缀路径 (如 https://domain.com/guide)。

2. 修复 结构化数据 (Schema.org / JSON-LD) [新增重点]

扫描 <script type="application/ld+json"> 块中的代码。

规则： 检查所有涉及 URL 的字段，包括但不限于：

"url": "..."

"@id": "..."

"mainEntityOfPage": "..."

BreadcrumbList 中的 "item": "..."

要求： 这些字段的值绝对不能包含 .html 后缀，必须与 Clean URL 保持一致。

3. 修复 内部链接 (HTML Tags)

扫描导航栏、页脚、侧边栏、文章推荐中的 <a> 标签。

规则： href 指向站内页面时，去掉 .html。

4. 修复 社交元数据 (Open Graph / Twitter)

检查 <meta property="og:url"> 和 <meta name="twitter:url">。

规则： content 必须是无后缀路径。

5. 修复 Sitemap 与 IndexNow 脚本

确保 sitemap.xml 和推送给必应的 urls.txt 列表里，所有链接都是无后缀的。
